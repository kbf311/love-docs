Luaのゲームフレームワーク **LÖVE (Love2D)** と、その有名ライブラリ **hump**（特に `hump.gamestate`）を使っている時に、`init`、`load`、`enter` の違いで混乱するのは「あるある」です！

どれも「初期化」っぽい雰囲気を持っていますが、**実行されるタイミングと目的**が全く異なります。

スッキリ整理できるように、それぞれの違いをまとめました。

---

## 3つの関数の決定的な違い

ざっくり一言で表すと、以下のようになります。

* **`init`**：「ゲーム起動時」に1回だけ。ステートがメモリに載った瞬間。
* **`load`**：**「使わない」**。LÖVE本来の関数であり、humpの画面遷移では役割を奪われます。
* **`enter`**：「その画面に切り替わった時」に毎回。

---

## 詳しい解説

### 1. `init(state)`

* **実行タイミング:** ゲームが起動し、そのステート（画面）のファイルが最初に読み込まれた（`require` された）タイミングで**1回だけ**実行されます。
* **使い道:** ゲーム全体でずっと使い回す画像やSEのロード、静的なデータの初期化など。
* **注意点:** 画面が切り替わるたびに実行されるわけではないので、「スコアの初期化」などをここに書くと、ゲームオーバーから戻ってきた時にスコアがリセットされません。

### 2. `load()`

* **実行タイミング:** LÖVE（本体）が起動した時に**1回だけ**。
* **使い道:** humpの管理下（`gamestate.switch` など）にある各ステート内では、基本的に**使いません**（定義しても無視されるか、意図しない挙動になります）。
* **正しい場所:** `main.lua` の中で、humpの最初のステートを呼び出すために使います。各画面のファイル内には書かないようにしましょう。

### 3. `enter(previous, ...)`

* **実行タイミング:** `gamestate.switch(変更先)` などによって、**その画面に切り替わった瞬間**に**毎回**実行されます。
* **使い道:** プレイヤーの位置のリセット、スコアの `0` クリア、そのステージ専用のBGMの再生など、「画面を開き直すたびにやり直したい処理」。
* **強力な機能:** 前の画面（`previous`）が何だったかを受け取れたり、`switch(state, arg1, arg2)` のように引数を渡して、それを受け取ったりできます（例: ステージ選択画面から「ステージ番号」を渡す）。

---

## コード例で見る流れ

例えば、タイトル画面（menu）**から**ゲーム本編（game）に切り替わるシーンを想像してみてください。

```lua
-- game.lua (ゲーム本編のステート)
local game = {}

-- 1. ゲーム起動時に1回だけ呼ばれる
function game:init()
    self.player_image = love.graphics.newImage("assets/player.png")
    print("game: init 完了（画像ロードなど）")
end

-- 2. この画面に切り替わるたびに呼ばれる
function game:enter(previous, score_to_beat)
    self.player_x = 100  -- プレイヤー位置を初期化
    self.score = 0       -- スコアをリセット
    print("game: enter 完了！前の画面は: " .. tostring(previous))
end

function game:update(dt) -- 省略
end
function game:draw() -- 省略
end

return game

```

### 実行される順番

1. **ゲーム起動時:** `game:init()` が走る。（画像が読み込まれる）
2. **タイトル画面から `gamestate.switch(game)` された時:** `game:enter()` が走る。（位置とスコアがリセットされてゲーム開始！）
3. **ゲームオーバーになって、またタイトルから戻ってきた時:** `game:init()` は**走らない**。`game:enter()` が**もう一度走る**。（位置とスコアが再びきれいにリセットされる！）

---

### まとめ

* **画像や音の読み込み** ＝ `init`
* **ゲームの状態リセット、画面切り替え時の演出開始** ＝ `enter`
* **各ステートファイル内での `load**` ＝ 不要（消してOK）

このように、「ゲーム中で何度も入り直す画面かどうか」を意識して使い分けると、バグが激減しますよ！
