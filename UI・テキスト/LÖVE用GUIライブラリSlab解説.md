# gemini response

**Slab** は、LÖVE (Love2D) でゲーム開発をする際に**めちゃくちゃ重宝する「イミディエイトモード（Immediate Mode）GUIライブラリ」**です。

C++開発者によく使われている有名な `Dear ImGui` というライブラリがあるのですが、Slab はまさに **「LÖVE 版の Dear ImGui」** と言える存在です。すべて純粋な Lua (Pure Lua) で書かれているため、導入が非常に簡単なのが特徴です。

---

## 🛠 Slab は何が便利なの？（主な用途）

ゲーム本編のUI（お洒落なタイトル画面やHPゲージなど）というよりは、**「ゲームを作るための開発者用ツール・デバッグ画面」** を作るのに最高に適しています。

* **開発効率の爆上げ:**
    マップエディタ、アイテムデータ編集、パラメータ調整用のスライダー、敵のスポーンデバッグ画面などが、数行のコードを足すだけで爆速で作れます。
* **配置の自動化:**
    通常のUIライブラリのように「X座標、Y座標、幅、高さを計算して……」と指定する必要がありません。コードを書いた順番にボタンやテキストが自動で並んでいきます。
* **豊富なコンポーネント:**
    ウィンドウのドラッグ＆ドロップ、折りたたみメニュー（Tree）、テキスト入力、タブ、カラーピッカー、メニューバー、ファイルのブラウズなど、デバッグに必要な機能が最初からほぼ揃っています。

## 💻 実際のコード例

どれくらい簡単に書けるか、基本のコードを見てみましょう。

```lua
local Slab = require 'Slab'

function love.load(args)
    -- Slabの初期化
    Slab.Initialize(args)
end

function love.update(dt)
    -- 毎フレームのアップデート
    Slab.Update(dt)

    -- 「デバッグ窓」というウィンドウを定義
    Slab.BeginWindow('DebugWindow', {Title = "デバッグメニュー", X = 20, Y = 20})
    
    -- テキストの表示
    Slab.Text("ゲームのパラメータ調整")
    
    -- ボタンの設置と、クリックされた時の判定が1行で書ける！
    if Slab.Button("プレイヤーのHPを回復") then
        player.hp = 100  -- ボタンが押された時の処理
    end
    
    Slab.EndWindow()
end

function love.draw()
    -- 最後にまとめて画面に描画
    Slab.Draw()
end
```

### 💡 イミディエイトモードの凄さ
普通のUIライブラリなら「ボタンオブジェクトを作って、リスナーを登録して…」と回帰的な処理（Retained Mode）が必要ですが、Slabのようなイミディエイトモードは **`if Slab.Button("...") then` と書くだけで「画面への描画」と「クリックされたかの判定」を同時に行ってくれます。** 状態管理が不要なので、コードが全くスパゲッティになりません。

## ⚠️ 使う上での注意点

* **ゲーム本編のUIには不向き:**
    デザインが基本的に「ツール用のグレースケールなウィンドウ」スタイルです。スタイル（色など）のカスタマイズはある程度可能ですが、アニメーションを多用した派手なゲーム画面や、スマホ向けのレスポンシブなUIを作りたい場合は、`SUIT` や `GÖST` など他のライブラリを使うか、自作したほうがスマートです。
* **動作の仕組み:**
    毎フレームUIを「再構築」しているため、あまりに巨大なデータを1画面に数万個同時に出そうとすると重くなることがあります（通常のデバッグ用途なら全く問題ありません）。
